# Release Notes: v0.3.0

**Release Date:** 2026-05-18

## Overview

polymarket-go v0.3.0 introduces a REST API server with automatic OpenAPI generation, GraphRAG for relationship-aware market retrieval, comprehensive resilience patterns, and structured error types for production-ready error handling.

## Highlights

- **REST API Server**: HTTP API with automatic OpenAPI spec generation using Huma + Chi
- **GraphRAG**: Relationship-aware retrieval with knowledge graph traversal for related markets
- **Resilience Patterns**: Retry with exponential backoff and circuit breakers for external services
- **Structured Errors**: Comprehensive error types with retryable detection and HTTP status mapping

## New Features

### REST API Server

A full-featured REST API server built with Huma and Chi:

```bash
# Start the server
polymarket-agent serve --port=8080 --with-news

# Access OpenAPI docs
curl http://localhost:8080/openapi.json
```

**Endpoints:**

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check |
| GET | `/markets` | List markets with filters |
| GET | `/markets/{conditionId}` | Get single market |
| GET | `/markets/{tokenId}/orderbook` | Order book |
| GET | `/markets/{tokenId}/price` | Token price |
| GET | `/news?q=query` | News search |
| GET | `/search?q=query` | Web search |
| POST | `/rag/markets/search` | Semantic market search |

OpenAPI documentation is automatically generated and available at `/docs`.

### GraphRAG Integration

Knowledge graph traversal for discovering related markets:

```go
import "github.com/grokify/polymarket-go/internal/rag"

// Create GraphRAG store
graphStore, _ := rag.NewGraphStore(rag.GraphStoreConfig{
    Graph: knowledgeGraph,
})

// Index markets with relationships
graphStore.IndexMarket(ctx, market)
graphStore.LinkSiblingMarkets(ctx, eventID, marketIDs)

// Find related markets via graph traversal
related, _ := graphStore.FindRelatedMarkets(ctx, marketID, 2)
```

**Graph Features:**

- Event → Market relationships
- Sibling market connections
- Category-based clustering
- Correlation edges between markets

### Resilience Patterns

Comprehensive resilience for external service calls:

```go
import "github.com/grokify/polymarket-go/internal/resilience"

// Retry with exponential backoff
result, err := resilience.Retry(ctx, resilience.RetryConfig{
    MaxAttempts:  3,
    InitialDelay: 100 * time.Millisecond,
    MaxDelay:     5 * time.Second,
    Multiplier:   2.0,
}, func(ctx context.Context) (string, error) {
    return client.GetMarkets(ctx, params)
})

// Circuit breaker
cb := resilience.NewCircuitBreaker(resilience.CircuitBreakerConfig{
    FailureThreshold: 5,
    ResetTimeout:     30 * time.Second,
})
err := cb.Execute(func() error {
    return externalAPICall()
})
```

### Structured Error Types

Production-ready error handling with type-safe errors:

```go
import perrors "github.com/grokify/polymarket-go/internal/errors"

// Create structured error
err := &perrors.APIError{
    Service:    "Polymarket",
    Operation:  "GetMarkets",
    StatusCode: 429,
}

// Check if retryable
if perrors.IsRetryable(err) {
    // Retry with backoff
}

// Get HTTP status code
status := perrors.HTTPStatusCode(err) // 429

// Use with errors.Is/As
if errors.Is(err, perrors.ErrRateLimited) {
    // Handle rate limiting
}
```

**Error Types:**

| Type | Use Case |
|------|----------|
| `APIError` | External API failures |
| `NetworkError` | Connection issues |
| `ValidationError` | Input validation |
| `ConfigurationError` | Config/init errors |
| `EmbeddingError` | Text embedding failures |
| `LLMError` | LLM provider errors |
| `CircuitOpenError` | Circuit breaker state |
| `RetryExhaustedError` | Max retries exceeded |
| `TimeoutError` | Operation timeouts |

### Ask LLM Command

New CLI command for ad-hoc LLM queries:

```bash
# Simple query
polymarket-agent ask "What are prediction markets?"

# With system prompt
polymarket-agent ask --system "You are a trading expert" "Explain limit orders"

# Pipe from stdin
echo "Explain Bitcoin" | polymarket-agent ask

# Use different model
polymarket-agent ask -m gpt-4 "Compare GPT vs Claude"
```

## Usage Examples

### Start REST API Server

```bash
# Basic server
polymarket-agent serve --port=8080

# With news search enabled
polymarket-agent serve --port=8080 --with-news

# With RAG search enabled
polymarket-agent serve --port=8080 --with-rag
```

### Use GraphRAG for Market Discovery

```go
// Index events and markets
for _, event := range events {
    graphStore.IndexEvent(ctx, event)
}
for _, market := range markets {
    graphStore.IndexMarket(ctx, market)
}

// Find markets related to a specific market
related, _ := graphStore.FindRelatedMarkets(ctx, "market-123", 2)
for _, r := range related {
    fmt.Printf("Related: %s (score: %.2f)\n", r.ID, r.Score)
}
```

### Resilient API Calls

```go
// Create circuit breaker for Polymarket API
cb := resilience.NewCircuitBreaker(resilience.CircuitBreakerConfig{
    Name:             "polymarket",
    FailureThreshold: 5,
    ResetTimeout:     30 * time.Second,
})

// Make resilient API call
result, err := resilience.Retry(ctx, resilience.DefaultRetryConfig(), func(ctx context.Context) (*Response, error) {
    return cb.ExecuteWithResult(func() (*Response, error) {
        return client.GetMarkets(ctx, params)
    })
})
```
